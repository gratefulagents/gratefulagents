# Account and appearance

**Settings** holds your personal preferences and configuration. Available pages depend on whether you use the web or desktop app.

## Open Settings

- Click **Settings** in the sidebar.
- Press <span className="kbd">⌘</span> <span className="kbd">,</span> on macOS or <span className="kbd">Ctrl</span> <span className="kbd">,</span> on Windows and Linux.
- Search for **Settings** in the command palette.

## Appearance

Choose **Light**, **Dark**, or **System**. System follows the operating-system appearance.

## Account

The account section shows the identity used for runs, credentials, and sharing. Select **Sign out** and confirm to end the current workspace session.

## Settings pages

| Page | Use it for |
| --- | --- |
| [Connection](./connection.md) | Desktop-only backend endpoint, Cloudflare Access, and locally saved workspaces. |
| [Desktop updates](./desktop-updates.md) | Desktop-only update token, checks, and installation. |
| [Credentials](./credentials.md) | Provider keys, OAuth sign-ins, GitHub token, and integration credentials. |
| [SOUL](./soul.md) | Your personal agent persona. |
| [Role models](./role-models.md) | Personal model overrides for available specialist roles. |
| [Git identity](./git-identity.md) | The name and email used to author commits in new runs. |

[Resources](./resources.md) are a separate sidebar destination. They hold reusable workspace configuration, including Skills and MCP servers; they are not Settings pages.

## Diagnostics

The Settings hub provides **Log locations** and **Report issue** links. Use logs when support asks for them. The report link opens the public GitHub issue form.
